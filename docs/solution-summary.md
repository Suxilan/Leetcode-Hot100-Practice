# 解法归纳（第一组）

## LC206 链表反转

- 题型: 链表指针操作
- 核心思路: prev, cur, nxt 三指针原地反转
- 关键步骤:
  1. 保存 nxt = cur.next
  2. cur.next = prev
  3. prev = cur, cur = nxt
- 易错点: 未提前保存 nxt 会导致链断
- 复杂度: 时间 O(n), 额外空间 O(1)

## LC283 移动零

- 题型: 双指针（快慢指针）
- 核心思路: slow 指向下一个非零该放置的位置，fast 扫描数组
- 关键步骤:
  1. 遍历 fast
  2. 若 nums[fast] 非零，交换 nums[slow], nums[fast]
  3. slow += 1
- 易错点: 不能破坏非零元素相对顺序
- 复杂度: 时间 O(n), 额外空间 O(1)

## LC3 无重复字符的最长子串

- 题型: 滑动窗口 + 哈希表
- 核心思路: 维护无重复窗口 [left, right]
- 关键步骤:
  1. 哈希记录字符最近出现位置
  2. 遇到重复字符时移动 left
  3. 持续更新答案长度
- 易错点: left 只能右移，不能回退
- 复杂度: 时间 O(n), 额外空间 O(k)

# 解法归纳（第二组）

## LC102 二叉树的层序遍历

- 题型: 二叉树 / 广度优先搜索 (BFS)
- 核心思路: 使用队列存储当前层的节点，每次循环处理一整层。
- 关键步骤:
  1. 特判根节点是否为空。
  2. 根节点入队。
  3. while 队列不空：记录当前 `level_len = len(queue)`。
  4. for 循环 `level_len` 次，节点出队，存入当前层结果，将其左右非空子节点入队。
- 易错点: 必须提前保存当前队列长度，否则会将下一层的节点混入当前层。
- 复杂度: 时间 O(n), 额外空间 O(n)（队列最大长度接近 n/2）

## LC236 二叉树的最近公共祖先

- 题型: 二叉树 / 深度优先搜索 (DFS)
- 核心思路: 后序遍历。利用子树返回的状态判断 target 节点位置。
- 关键步骤:
  1. Base case: 若 `root` 为 null、`p` 或 `q`，直接返回 `root`。
  2. 递归查找左子树和右子树：`left = dfs(root.left)`, `right = dfs(root.right)`。
  3. 若 `left` 和 `right` 都有返回值，说明 `p` 和 `q` 异侧，`root` 就是 LCA。
  4. 否则返回那个非空的返回值（同侧情况）。
- 易错点: `p` 本身是 `q` 的祖先的情况（由 Base case 自然兼容处理）。
- 复杂度: 时间 O(n), 额外空间 O(h)（递归调用栈深度）

## LC124 二叉树中的最大路径和

- 题型: 二叉树 / 树形 DP (DFS 后序遍历)
- 核心思路: 对于每个节点，计算以它为最高点的“拐弯”路径和，同时向上传递“单边”最大路径和。
- 关键步骤:
  1. 初始化全局变量 `max_sum` 为负无穷。
  2. `dfs(node)`：计算以 `node` 为起点的**单向向下**最大路径和。
     若子树向下总和为负，则**舍弃**(与 0 取 max)。
  3. 当前节点为拐点的最大路径和 = `node.val + left_max + right_max`，用该值去更新全局 `max_sum`。
  4. 返回值 `node.val + max(left_max, right_max)`，供父节点使用。
  5. 启动 `dfs(root)`，返回 `max_sum`。
- 易错点: `max_sum` 更新用双边(左右子树都加)，但函数的 `return` 只能带走单边(只能走一侧)。节点值为负数时，不能直接赋 0。
- 复杂度: 时间 O(n), 额外空间 O(h)

# 解法归纳（第三组）

## LC70 爬楼梯

- 题型: 动态规划 / 线性递推
- 核心思路: 到达第 i 阶的方法数由前两阶转移而来。
- 关键步骤:
  1. 定义 `dp[i]` 表示到达第 i 阶的方法数。
  2. 转移方程: `dp[i] = dp[i - 1] + dp[i - 2]`。
  3. 初始条件: `dp[1] = 1`, `dp[2] = 2`。
  4. 可做空间优化，只保留前两项。
- 易错点: `n <= 2` 的边界要单独处理。
- 复杂度: 时间 O(n), 额外空间 O(1)（滚动变量）

## LC322 零钱兑换

- 题型: 动态规划 / 完全背包（最小值）
- 核心思路: `dp[x]` 表示凑成金额 x 的最少硬币数。
- 关键步骤:
  1. 初始化 `dp[0] = 0`，其余为正无穷。
  2. 遍历金额 `x`，尝试每个硬币 `c`。
  3. 当 `x >= c` 时，更新 `dp[x] = min(dp[x], dp[x - c] + 1)`。
  4. 若最终 `dp[amount]` 仍为无穷，输出 -1。
- 易错点: 初始化不要用 0；完全背包要允许同一硬币重复使用。
- 复杂度: 时间 O(n * amount), 额外空间 O(amount)

## LC300 最长递增子序列

- 题型: 动态规划 / 贪心 + 二分
- 核心思路: 维护数组 `tails`，`tails[k]` 表示长度为 k+1 的递增子序列最小尾值。
- 关键步骤:
  1. 依次遍历每个数 x。
  2. 在 `tails` 中二分查找第一个 `>= x` 的位置 pos。
  3. 若 pos 存在则替换，否则追加 x。
  4. `len(tails)` 即 LIS 长度。
- 易错点: 递增是严格 `<`，二分应找第一个 `>= x`。
- 复杂度: 时间 O(n log n), 额外空间 O(n)
