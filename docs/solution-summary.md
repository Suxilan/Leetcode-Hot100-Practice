# 解法归纳（第一组）

## LC206 链表反转

- 题型: 链表指针操作
- 核心思路: prev, cur, nxt 三指针原地反转
- 关键步骤:
  1. 保存 nxt = cur.next
  2. cur.next = prev
  3. prev = cur, cur = nxt
- 易错点: 未提前保存 nxt 会导致链断
- 复杂度: 时间 O(n), 额外空间 O(1)

## LC283 移动零

- 题型: 双指针（快慢指针）
- 核心思路: slow 指向下一个非零该放置的位置，fast 扫描数组
- 关键步骤:
  1. 遍历 fast
  2. 若 nums[fast] 非零，交换 nums[slow], nums[fast]
  3. slow += 1
- 易错点: 不能破坏非零元素相对顺序
- 复杂度: 时间 O(n), 额外空间 O(1)

## LC3 无重复字符的最长子串

- 题型: 滑动窗口 + 哈希表
- 核心思路: 维护无重复窗口 [left, right]
- 关键步骤:
  1. 哈希记录字符最近出现位置
  2. 遇到重复字符时移动 left
  3. 持续更新答案长度
- 易错点: left 只能右移，不能回退
- 复杂度: 时间 O(n), 额外空间 O(k)

# 解法归纳（第二组）

## LC102 二叉树的层序遍历

- 题型: 二叉树 / 广度优先搜索 (BFS)
- 核心思路: 使用队列存储当前层的节点，每次循环处理一整层。
- 关键步骤:
  1. 特判根节点是否为空。
  2. 根节点入队。
  3. while 队列不空：记录当前 `level_len = len(queue)`。
  4. for 循环 `level_len` 次，节点出队，存入当前层结果，将其左右非空子节点入队。
- 易错点: 必须提前保存当前队列长度，否则会将下一层的节点混入当前层。
- 复杂度: 时间 O(n), 额外空间 O(n)（队列最大长度接近 n/2）

## LC236 二叉树的最近公共祖先

- 题型: 二叉树 / 深度优先搜索 (DFS)
- 核心思路: 后序遍历。利用子树返回的状态判断 target 节点位置。
- 关键步骤:
  1. Base case: 若 `root` 为 null、`p` 或 `q`，直接返回 `root`。
  2. 递归查找左子树和右子树：`left = dfs(root.left)`, `right = dfs(root.right)`。
  3. 若 `left` 和 `right` 都有返回值，说明 `p` 和 `q` 异侧，`root` 就是 LCA。
  4. 否则返回那个非空的返回值（同侧情况）。
- 易错点: `p` 本身是 `q` 的祖先的情况（由 Base case 自然兼容处理）。
- 复杂度: 时间 O(n), 额外空间 O(h)（递归调用栈深度）

## LC124 二叉树中的最大路径和

- 题型: 二叉树 / 树形 DP (DFS 后序遍历)
- 核心思路: 对于每个节点，计算以它为最高点的“拐弯”路径和，同时向上传递“单边”最大路径和。
- 关键步骤:
  1. 初始化全局变量 `max_sum` 为负无穷。
  2. `dfs(node)`：计算以 `node` 为起点的**单向向下**最大路径和。
     若子树向下总和为负，则**舍弃**(与 0 取 max)。
  3. 当前节点为拐点的最大路径和 = `node.val + left_max + right_max`，用该值去更新全局 `max_sum`。
  4. 返回值 `node.val + max(left_max, right_max)`，供父节点使用。
  5. 启动 `dfs(root)`，返回 `max_sum`。
- 易错点: `max_sum` 更新用双边(左右子树都加)，但函数的 `return` 只能带走单边(只能走一侧)。节点值为负数时，不能直接赋 0。
- 复杂度: 时间 O(n), 额外空间 O(h)

# 解法归纳（第三组）

## LC70 爬楼梯

- 题型: 动态规划 / 线性递推
- 核心思路: 到达第 i 阶的方法数由前两阶转移而来。
- 关键步骤:
  1. 定义 `dp[i]` 表示到达第 i 阶的方法数。
  2. 转移方程: `dp[i] = dp[i - 1] + dp[i - 2]`。
  3. 初始条件: `dp[1] = 1`, `dp[2] = 2`。
  4. 可做空间优化，只保留前两项。
- 易错点: `n <= 2` 的边界要单独处理。
- 复杂度: 时间 O(n), 额外空间 O(1)（滚动变量）

## LC322 零钱兑换

- 题型: 动态规划 / 完全背包（最小值）
- 核心思路: `dp[x]` 表示凑成金额 x 的最少硬币数。
- 关键步骤:
  1. 初始化 `dp[0] = 0`，其余为正无穷。
  2. 遍历金额 `x`，尝试每个硬币 `c`。
  3. 当 `x >= c` 时，更新 `dp[x] = min(dp[x], dp[x - c] + 1)`。
  4. 若最终 `dp[amount]` 仍为无穷，输出 -1。
- 易错点: 初始化不要用 0；完全背包要允许同一硬币重复使用。
- 复杂度: 时间 O(n * amount), 额外空间 O(amount)

## LC300 最长递增子序列

- 题型: 动态规划 / 贪心 + 二分
- 核心思路: 维护数组 `tails`，`tails[k]` 表示长度为 k+1 的递增子序列最小尾值。
- 关键步骤:
  1. 依次遍历每个数 x。
  2. 在 `tails` 中二分查找第一个 `>= x` 的位置 pos。
  3. 若 pos 存在则替换，否则追加 x。
  4. `len(tails)` 即 LIS 长度。
- 易错点: 递增是严格 `<`，二分应找第一个 `>= x`。
- 复杂度: 时间 O(n log n), 额外空间 O(n)

# 解法归纳（第四组）

## LC46 全排列

- 题型: 回溯
- 核心思路: 构建一棵深度为 n 的决策树，每层选择一个未使用元素。
- 关键步骤:
  1. 维护 `path` 和 `used[]`。
  2. 当 `len(path) == n` 时收集一个答案。
  3. 遍历所有下标，未使用则选中 -> 递归 -> 回溯撤销。
- 易错点: 回溯时必须同时撤销 `path` 和 `used` 状态。
- 复杂度: 时间 O(n * n!), 额外空间 O(n)

## LC78 子集

- 题型: 回溯
- 核心思路: 每个元素都有“选/不选”两种决策，可用起始下标控制组合扩展。
- 关键步骤:
  1. 先把当前 `path` 加入答案。
  2. 从 `start` 开始枚举后续元素。
  3. 选中当前元素，递归 `dfs(i + 1)`，然后撤销。
- 易错点: 子集题先收集再向下扩展，和全排列收集时机不同。
- 复杂度: 时间 O(n * 2^n), 额外空间 O(n)

## LC200 岛屿数量

- 题型: 矩阵 DFS/BFS
- 核心思路: 扫描网格，遇到未访问陆地就触发一次 DFS/BFS 并计数 +1。
- 关键步骤:
  1. 双层循环扫描网格。
  2. 遇到 `'1'` 执行 flood fill，把整块连通陆地标记。
  3. 每触发一次 flood fill，岛屿数加 1。
- 易错点: 访问标记要及时，避免重复搜索导致超时或重复计数。
- 复杂度: 时间 O(mn), 额外空间 O(mn)（递归栈/队列最坏）

## LC54 螺旋矩阵

- 题型: 数组模拟 / 边界控制
- 核心思路: 用四个边界 `top/bottom/left/right` 分圈层遍历。
- 关键步骤:
  1. 从左到右扫 `top`，`top += 1`。
  2. 从上到下扫 `right`，`right -= 1`。
  3. 若边界仍合法，从右到左扫 `bottom`，`bottom -= 1`。
  4. 若边界仍合法，从下到上扫 `left`，`left += 1`。
- 易错点: 每一步缩边界后都要判断是否越界，避免重复访问。
- 复杂度: 时间 O(mn), 额外空间 O(1)（不计答案数组）

## LC146 LRU 缓存

- 题型: 设计 / 哈希表 + 双向链表（或 OrderedDict）
- 核心思路: 哈希 O(1) 定位节点，链表 O(1) 调整最近使用顺序。
- 关键步骤:
  1. `get(key)`：命中后节点移动到头部并返回值。
  2. `put(key, value)`：若已存在则更新并移动到头部。
  3. 若容量超限，淘汰尾部最久未使用节点。
- 易错点: 更新已存在 key 时，不应增加 size；淘汰时哈希与链表要同步删除。
- 复杂度: `get/put` 均摊 O(1)，额外空间 O(capacity)

# 解法归纳（第五组）

## LC42 接雨水

- 题型: 双指针
- 核心思路: 用 `left/right` 和 `left_max/right_max` 收敛，低的一侧决定当前可接水量。
- 关键步骤:
  1. 初始化 `left=0`, `right=n-1`, `left_max=0`, `right_max=0`。
  2. 若 `height[left] < height[right]`，处理左侧并移动 `left`。
  3. 否则处理右侧并移动 `right`。
  4. 每步累加 `max(0, left_max-height[left])` 或右侧对应值。
- 易错点: 更新 max 与计算接水顺序不要写反。
- 复杂度: 时间 O(n), 额外空间 O(1)

## LC739 每日温度

- 题型: 单调栈（递减栈）
- 核心思路: 栈里放下标，保持对应温度递减；当前更高温度可结算栈顶答案。
- 关键步骤:
  1. 遍历每一天 i。
  2. 当 `T[i] > T[stack[-1]]` 时，弹栈并计算等待天数。
  3. 当前下标入栈。
- 易错点: 栈必须存下标，答案是下标差，不是温差。
- 复杂度: 时间 O(n), 额外空间 O(n)

## LC84 柱状图中最大的矩形

- 题型: 单调栈（递增栈）
- 核心思路: 维护递增高度下标栈，遇到更矮柱子时结算被截断柱子的最大面积。
- 关键步骤:
  1. 在 heights 两端补 0（哨兵），统一结算逻辑。
  2. 遍历下标 i，若当前高度小于栈顶高度，持续弹栈计算面积。
  3. 宽度由“当前下标 i 与新栈顶下标”决定。
- 易错点: 栈里存下标不是高度，因为要计算宽度边界。
- 复杂度: 时间 O(n), 额外空间 O(n)

## LC239 滑动窗口最大值

- 题型: 滑动窗口 + 单调队列（双端队列）
- 核心思路: 队列存下标并保持对应值递减，队首永远是窗口最大值。
- 关键步骤:
  1. 新元素进入前，弹出队尾所有小于等于它的元素下标。
  2. 新下标入队。
  3. 若队首下标已离开窗口左边界，弹出队首。
  4. 当窗口成型后记录队首对应值。
- 易错点: 先处理队尾递减性，再处理队首过期。
- 复杂度: 时间 O(n), 额外空间 O(k)

## LC33 搜索旋转排序数组

- 题型: 二分查找变体
- 核心思路: 每轮先判定哪一半有序，再判断 target 是否落在有序半区。
- 关键步骤:
  1. 常规二分取 mid。
  2. 若左半有序（`nums[left] <= nums[mid]`），判断 target 是否在左半。
  3. 否则右半有序，判断 target 是否在右半。
  4. 不在有序半区就去另一半。
- 易错点: 边界判定的等号容易写错，尤其是 `<=` 和 `<` 的组合。
- 复杂度: 时间 O(log n), 额外空间 O(1)

# 解法归纳（第六组）

## LC15 三数之和

- 题型: 排序 + 双指针
- 核心思路: 固定第一个数后，在右侧区间做两数之和，利用有序性收缩指针。
- 关键步骤:
  1. 先排序。
  2. 枚举 `i` 作为第一个数，跳过重复 `nums[i]`。
  3. 用 `l/r` 查找 `nums[l] + nums[r] = -nums[i]`。
  4. 命中后同时跳过左右重复值，彻底去重。
- 易错点: 去重不完整会产生重复三元组。
- 复杂度: 时间 O(n^2), 额外空间 O(1)（不计输出）

## LC56 合并区间

- 题型: 排序 + 贪心
- 核心思路: 按左端点排序后线性扫描，维护当前合并区间。
- 关键步骤:
  1. 区间按起点升序排序。
  2. 若新区间起点 `<=` 当前区间终点，则合并更新终点。
  3. 否则把当前区间加入答案并切换到新区间。
  4. 扫描结束后别忘了加入最后一个区间。
- 易错点: 是否可合并的条件是 `<=`，不是 `<`。
- 复杂度: 时间 O(n log n), 额外空间 O(1)（不计排序栈/输出）

## LC560 和为 K 的子数组

- 题型: 前缀和 + 哈希表
- 核心思路: 若 `pre[j] - pre[i] = k`，则以 `j` 结尾的有效子数组数量取决于历史前缀和 `pre[j]-k` 出现次数。
- 关键步骤:
  1. 维护前缀和 `pre` 和计数哈希 `cnt`。
  2. 初始 `cnt[0] = 1`，处理从下标 0 开始的子数组。
  3. 遍历每个数更新 `pre`，累加 `cnt[pre-k]` 到答案。
  4. 再更新 `cnt[pre] += 1`。
- 易错点: `cnt[0]=1` 常被漏掉，导致少算前缀刚好为 k 的情况。
- 复杂度: 时间 O(n), 额外空间 O(n)

## LC215 数组中的第K个最大元素

- 题型: 选择算法（QuickSelect）/ 堆
- 核心思路: 不做全排序，目标是定位第 `n-k` 小元素。
- 关键步骤:
  1. QuickSelect：分区后只递归目标所在半边。
  2. 或最小堆：维护大小为 k 的堆，最终堆顶就是第 k 大。
  3. 面试优先会手写 QuickSelect，复杂度更优。
- 易错点: 下标转换 `kth_largest -> index = n-k` 容易写错。
- 复杂度: QuickSelect 平均 O(n)；最小堆 O(n log k)

## LC20 有效的括号

- 题型: 栈
- 核心思路: 遇到左括号入栈，遇到右括号必须与栈顶匹配。
- 关键步骤:
  1. 建立右括号到左括号的映射。
  2. 扫描字符：左括号入栈；右括号检查栈顶匹配。
  3. 任一步不匹配立即 false，最终栈空才 true。
- 易错点: 栈空时直接遇到右括号要立即 false。
- 复杂度: 时间 O(n), 额外空间 O(n)

# 解法归纳（第七组）

## LC4 寻找两个正序数组的中位数

- 题型: 二分查找 (Hard)
- 核心思路: 不合并数组，而是对较短的数组进行二分查找“划分线”，使得左半边最大值小于等于右半边最小值。
- 关键步骤:
  1. 保证 `nums1` 是较短的数组，在 `[0, m]` 范围内二分划分位置 `i`，`nums2` 的划分位置 `j = (m + n + 1) // 2 - i`。
  2. 比较 `nums1[i-1]` 和 `nums2[j]` 以及 `nums2[j-1]` 和 `nums1[i]` 决定二分方向。
  3. 越界时用 `-inf` 和 `inf` 当作虚拟边界处理。
- 易错点: 数组长度奇偶性带来的中位数下标偏移；越界条件判断极为苛刻。
- 复杂度: 时间 O(log(min(m, n))), 额外空间 O(1)

## LC31 下一个排列

- 题型: 数组模拟 / 原地修改
- 核心思路: 通过寻找靠右侧的“较小数”和“较大数”交换，再翻转后缀实现最小幅度的变大。
- 关键步骤:
  1. 从后向前遍历，找到第一个满足 `nums[i] < nums[i+1]` 的下标 `i`。
  2. 如果找得到 `i`，再从后向前找第一个满足 `nums[j] > nums[i]` 的下标 `j`，交换 `nums[i]` 和 `nums[j]`。
  3. 将 `i+1` 之后的所有元素翻转（变为升序，保证变大的幅度最小）。
- 易错点: 数组原本就是完全降序（最大排列）时，直接全部翻转即可。
- 复杂度: 时间 O(n), 额外空间 O(1)

## LC148 排序链表

- 题型: 链表 / 归并排序
- 核心思路: 满足 O(n log n) 时间和 O(1) 空间的最优解是自底向上的归并排序，但面试常考自顶向下的递归归并（辅助空间 O(log n)）。
- 关键步骤:
  1. 找中点：使用快慢指针，快指针初始为 `head.next`，慢指针为 `head`，以便偶数节点时从中偏左断开。
  2. 断链：保存 `mid = slow.next`，并打断 `slow.next = None`。
  3. 合并：递归排序两个子链表，然后应用标准的合并两个有序链表流程（LC21）。
- 易错点: 中点查找如果不把 fast 错开一位，会导致 2 个节点时陷入无限递归死循环。
- 复杂度: 时间 O(n log n), 额外空间 O(log n) (递归栈) 或 O(1) (迭代写法)
